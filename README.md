## 📌 커밋 컨벤션

각 태그를 이용하여 어떤 내용이 변경되었는지를 나타내는 규칙입니다.

- `[FEAT]` : 새로운 기능 추가  
- `[DESIGN]` : UI 수정  
- `[REFACT]` : 코드 유지보수/가독성 향상  
- `[FIX]` : 버그 에러 수정  

---

## 📐 코드 컨벤션

코드에서 이름을 지정할 때의 케이스 규칙입니다.

- `PascalCase` : 컴포넌트, 클래스명 등  
- `camelCase` : 변수, 함수명 등  
- `UPPER_CASE` : 상수
- 'kebab-case' : 파일,폴더 (컴포넌트 파일만 예외로  PascalCase러 설정)
